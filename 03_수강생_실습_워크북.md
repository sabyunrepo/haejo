# 수강생 실습 워크북 — 따라하기 가이드

> 이 문서는 **수강생 본인이 손에 쥐고 한 단계씩 따라하는 실습서**입니다.
> 강의 중·강의 후·복습 시 모두 활용 가능합니다.
> 막힐 때는 **"💡 막힐 때"** 박스를 먼저 보세요.

---

## 📌 사용 규칙

| 표기 | 의미 |
|---|---|
| ☐ | 체크박스 — 끝났으면 ✓ 표시 |
| `(터미널)` | PowerShell에 입력 |
| `(Claude Code)` | Claude Code Desktop 채팅창에 입력 |
| `(브라우저)` | Chrome/Edge 등에서 작업 |
| 💡 막힐 때 | 자주 발생하는 문제 + 해결법 |
| 📎 응용 | 본인 서비스로 변형하고 싶을 때 참고 |

---

## ⚠️ 가장 중요한 한 가지 — "에러 = 단서"

**에러 메시지가 빨갛게 떠도 당황하지 마세요.** 정상입니다.

대응 순서는 항상 같습니다:
1. **에러 메시지 전체를 마우스로 드래그해서 복사**
2. Claude Code Desktop에 붙여넣고 → **"이 에러 무슨 뜻이야? 어떻게 고쳐?"**
3. Claude가 알려주는 대로 수정
4. 다시 실행

이 한 가지 패턴이 강의의 90%입니다.

# 📅 Day 2 — 데이터 + AI + 인터넷 공개

## 💾 4차시 — Supabase로 회원가입 + 메모 저장 (60분)

#### ☐ 4-1. Supabase 프로젝트 만들기

1. (브라우저) [supabase.com](https://supabase.com) → "Start your project" → GitHub 로그인
2. "New Project" 클릭
3. 입력:
   - Name: `my-notes`
   - **Database Password**: 강한 비번 + **꼭 메모!** (분실 시 복구 불가)
   - Region: `Northeast Asia (Seoul)`
   - Plan: Free
4. 1~2분 대기

#### ☐ 4-2. memos 테이블 만들기

1. 왼쪽 메뉴 → "Table Editor" 또는 "Database" → "Tables"
2. "New table" → 이름 `memos`
3. **"Enable Row Level Security (RLS)" 켜진 상태 유지**
4. 컬럼 추가:

| 컬럼 이름 | 타입 | 기본값 |
|---|---|---|
| id | uuid | `gen_random_uuid()` |
| user_id | uuid | (없음) |
| content | text | (없음) |
| summary | text | (없음, nullable) |
| created_at | timestamptz | `now()` |

5. Save

#### ☐ 4-3. API 키 가져오기

1. 왼쪽 메뉴 → "Settings" → "API"
2. 다음 두 값을 복사 (별도 메모장에 임시 저장):
   - **Project URL** (`https://xxxxx.supabase.co`)
   - **anon public key** (긴 문자열)
3. ⚠️ **service_role key는 절대 만지지 말기**

#### ☐ 4-4. .env.local 파일 만들기

```
(Claude Code)
my-notes 폴더에 .env.local 파일을 만들어 줘.
내용은 다음과 같아:

NEXT_PUBLIC_SUPABASE_URL=여기에_프로젝트_URL_붙여넣음
NEXT_PUBLIC_SUPABASE_ANON_KEY=여기에_anon_키_붙여넣음

그리고 .gitignore에 .env.local이 이미 포함되어 있는지 확인해 줘.
```

#### ☐ 4-5. Supabase 클라이언트 연결

```
(Claude Code)
my-notes 프로젝트에 Supabase를 연결해 줘.

1. @supabase/supabase-js 패키지를 설치
2. lib/supabaseClient.ts 파일을 만들어서 거기에 클라이언트 초기화 코드 작성
3. .env.local의 NEXT_PUBLIC_SUPABASE_URL과 NEXT_PUBLIC_SUPABASE_ANON_KEY를 사용
4. 한국어 주석 달기
```

#### ☐ 4-6. 회원가입 + 로그인 + CRUD 만들기

```
(Claude Code)
my-notes에 다음 기능을 추가해 줘. 모두 Supabase로 동작하게.

1. 회원가입 / 로그인 페이지 (이메일 + 비밀번호 방식)
   - 로그인 후엔 메모 페이지로 이동
   - 로그아웃 버튼도 있음

2. 메모 CRUD (현재 page.tsx 수정)
   - 로그인하지 않으면 로그인 페이지로 자동 이동
   - 본인의 메모만 보임
   - 메모 작성: textarea + 추가 버튼 → memos 테이블에 저장
   - 메모 목록: 내가 쓴 메모만 최신순으로 표시
   - 각 메모마다 삭제 버튼

3. Supabase RLS 정책 SQL도 같이 알려줘
   - 본인 user_id의 행만 SELECT/INSERT/DELETE 가능

4. 한국어 주석 풍부하게
5. Tailwind로 깔끔하게
```

#### ☐ 4-7. RLS 정책 적용

Supabase → "SQL Editor" → New query → Claude가 알려준 SQL 붙여넣고 "Run"

예시:
```sql
CREATE POLICY "본인 메모 조회"
ON memos FOR SELECT
USING (auth.uid() = user_id);

CREATE POLICY "본인 메모 추가"
ON memos FOR INSERT
WITH CHECK (auth.uid() = user_id);

CREATE POLICY "본인 메모 삭제"
ON memos FOR DELETE
USING (auth.uid() = user_id);
```

#### ☐ 4-8. 이메일 인증 끄기 (학습 편의)

Supabase → Authentication → Providers → Email → "Confirm email" 체크 해제

#### ☐ 4-9. 동작 테스트

(터미널) — `.env.local` 변경 후 반드시 재시작:

테스트:

- ☐ 메모 작성 → 목록에 표시
- ☐ 새로고침 후에도 메모가 남아 있음
- ☐ 다른 계정으로 새로 가입 → **첫 계정의 메모가 안 보임** (RLS 검증)

#### ☐ 4-10. GitHub 푸시


## 🤖 5차시 — Gemini API로 AI 요약 추가 (60분)

#### ☐ 5-1. Gemini API 키 발급

1. (브라우저) [aistudio.google.com](https://aistudio.google.com) → Google 로그인
2. "Get API key" → "Create API key" → "Create API key in new project"
3. **키 복사 → 메모! (다시 못 봄)**

#### ☐ 5-2. .env.local에 추가

(VS Code 또는 Claude Code) `.env.local` 끝에 한 줄 추가:

```
GEMINI_API_KEY=여기에_방금_발급한_키
```

⚠️ `NEXT_PUBLIC_`는 **붙이지 않습니다** (서버에서만 사용).

(터미널) `npm run dev` **재시작** (`Ctrl+C` → `npm run dev`).

#### ☐ 5-3. AI 요약 기능 만들기

```
(Claude Code)
my-notes에 'AI 요약' 기능을 추가해 줘.

1. Google Generative AI SDK 설치 (@google/generative-ai)

2. API Route 만들기: app/api/summarize/route.ts
   - POST 요청을 받음
   - 요청 본문에 { content: 메모 내용 } 형식
   - 환경변수 GEMINI_API_KEY를 사용
   - gemini-1.5-flash 모델로 요약
   - 프롬프트: "다음 메모를 3줄 이내로 한국어로 간결히 요약해줘"
   - 결과를 { summary: "..." } 형식 JSON으로 반환

3. 메모 목록 화면(app/page.tsx) 수정
   - 각 메모 카드에 'AI 요약' 버튼 추가
   - 누르면 위 API Route를 호출
   - 받아온 요약을 memos 테이블의 summary 컬럼에 저장
   - 화면의 그 메모 아래에 요약 결과 표시
```

# 🔧 막힐 때 — 에러 패턴 모음

## 환경
| 에러 | 해결 |
|---|---|
| `'node' is not recognized` | PowerShell **새 창**, 안 되면 PC 재시작 |
| `'git' is not recognized` | 새 창 열기 |
| 권한 거부 | 관리자 PowerShell 또는 다른 폴더 시도 |

## Next.js
| 에러 | 해결 |
|---|---|
| `Module not found` | `npm install` 다시 |
| 화면 안 변함 | `Ctrl+F5` 강제 새로고침 |
| `npm run dev` 후 페이지 안 뜸 | 터미널 메시지에서 진짜 포트 확인 (3000 점유 시 3001로) |

## Supabase
| 에러 | 해결 |
|---|---|
| Invalid API key | `.env.local` 확인 + `npm run dev` 재시작 |
| Row level security violated | RLS 정책 SQL 다시 실행 |
| Email not confirmed | Authentication → Email → Confirm email 끄기 |

## Gemini
| 에러 | 해결 |
|---|---|
| 401 Unauthorized | 키 오타·공백 / `npm run dev` 재시작 |
| 429 Too Many Requests | 1분 후 재시도 |
| 응답이 영어 | "반드시 한국어로" 프롬프트에 추가 |

## Vercel
| 에러 | 해결 |
|---|---|
| Build failed | 마지막 빨간 글자를 Claude Code에 복붙 |
| Root Directory 잘못됨 | Project → Settings → Build & Development → Root Directory 변경 |
| 운영에서 Auth 실패 | Supabase Auth Redirect URL 추가 안 했을 가능성 99% |

> 💡 **모든 에러의 만능 답**:
> **"에러 메시지 전체 → 복사 → Claude Code에 붙여넣기 → '이거 무슨 에러야? 어떻게 고쳐?'"**
> 강의 내내 이 한 패턴이면 90% 해결됩니다.
